# Entrada OS - Структура проекта

## Архитектура системы

```
┌─────────────────────────────────────┐
│         Shell (shell.c)             │  ← Пользовательский интерфейс
├─────────────────────────────────────┤
│      System Calls (syscall.c)       │  ← API для приложений
├─────────────────────────────────────┤
│  Process Manager (process.c)        │  ← Управление процессами
├─────────────────────────────────────┤
│  Memory Manager (memory.c)          │  ← Управление памятью
├─────────────────────────────────────┤
│  Drivers: Keyboard, Timer, Screen   │  ← Драйверы устройств
├─────────────────────────────────────┤
│  Interrupt Handler (idt.c)          │  ← Обработка прерываний
├─────────────────────────────────────┤
│  GDT (gdt.c)                        │  ← Сегментация памяти
├─────────────────────────────────────┤
│  Kernel Core (kernel.c)             │  ← Ядро системы
├─────────────────────────────────────┤
│  Multiboot Loader (multiboot.asm)   │  ← Загрузчик
└─────────────────────────────────────┘
```

## Файлы проекта

### Загрузка и инициализация
- **multiboot.asm** - Multiboot заголовок, точка входа, инициализация стека
- **kernel.c** - Главная функция ядра, инициализация подсистем
- **linker.ld** - Скрипт компоновщика, определяет расположение секций в памяти

### Управление памятью
- **gdt.c / gdt.asm** - Global Descriptor Table, сегментация памяти
- **memory.c / memory.h** - Динамическое выделение памяти (kmalloc/kfree)

### Прерывания и драйверы
- **idt.c / idt.h** - Interrupt Descriptor Table, обработка прерываний
- **interrupt.asm** - Низкоуровневые обработчики прерываний (ISR/IRQ)
- **keyboard.c / keyboard.h** - Драйвер клавиатуры PS/2
- **timer.c / timer.h** - Программируемый таймер (PIT)
- **screen.c / screen.h** - VGA текстовый режим 80x25

### Процессы и системные вызовы
- **process.c / process.h** - Управление процессами, планировщик
- **syscall.c / syscall.h** - Системные вызовы (write, read, exit, getpid)

### Пользовательский интерфейс
- **shell.c / shell.h** - Командная оболочка

### Утилиты
- **string.c** - Функции работы со строками (strlen, strcmp, strcpy, memset, memcpy)
- **types.h** - Определения базовых типов данных
- **kernel.h** - Общие определения и функции ядра

### Сборка
- **Makefile** - Автоматическая сборка проекта
- **grub.cfg** - Конфигурация загрузчика GRUB
- **build.sh** - Скрипт сборки для Linux/Mac
- **Dockerfile** - Контейнер для сборки

## Карта памяти

```
0x00000000 - 0x000003FF  : Real Mode IVT (не используется)
0x00000400 - 0x000004FF  : BIOS Data Area
0x00000500 - 0x00007BFF  : Свободно
0x00007C00 - 0x00007DFF  : Bootloader (не используется в Multiboot)
0x00007E00 - 0x0009FFFF  : Свободно
0x000A0000 - 0x000BFFFF  : Video RAM
0x000C0000 - 0x000FFFFF  : BIOS ROM
0x00100000 - 0x001FFFFF  : Kernel Code & Data (1MB)
0x00100000 - 0x001FFFFF  : Heap (1MB)
0x00200000+              : Свободно для расширения
```

## Поток выполнения

### 1. Загрузка (GRUB)
```
GRUB → multiboot.asm:kernel_entry
  ↓
Инициализация стека (ESP = stack_top)
  ↓
Вызов kernel_main()
```

### 2. Инициализация ядра
```
kernel_main() {
  screen_clear()
  gdt_init()        // Настройка сегментации
  idt_init()        // Настройка прерываний
  memory_init()     // Инициализация heap
  keyboard_init()   // Инициализация драйвера клавиатуры
  timer_init(50)    // Таймер 50 Hz
  process_init()    // Система процессов
  syscall_init()    // Системные вызовы
  shell_main()      // Запуск shell
}
```

### 3. Работа Shell
```
shell_main() {
  while(1) {
    print "entrada> "
    read command
    execute command
  }
}
```

### 4. Обработка прерываний
```
Прерывание → interrupt.asm:irq_common_stub
  ↓
Сохранение регистров (pusha)
  ↓
irq_handler() в idt.c
  ↓
Вызов зарегистрированного обработчика
  ↓
Восстановление регистров (popa)
  ↓
Возврат (iret)
```

## Системные вызовы

```c
// Вызов из пользовательского кода
int syscall(int num, void* arg1, void* arg2) {
    int ret;
    asm volatile("int $0x80" 
                 : "=a"(ret) 
                 : "a"(num), "b"(arg1), "c"(arg2));
    return ret;
}

// Доступные вызовы:
// 1 - SYSCALL_WRITE  : вывод строки
// 2 - SYSCALL_READ   : чтение ввода
// 3 - SYSCALL_EXIT   : завершение процесса
// 4 - SYSCALL_GETPID : получить PID
```

## Расширение системы

### Добавление нового драйвера

1. Создать файлы `driver.c` и `driver.h`
2. Реализовать функции инициализации и обработки
3. Зарегистрировать обработчик прерывания в `idt.c`
4. Вызвать `driver_init()` в `kernel_main()`
5. Добавить в Makefile

### Добавление новой команды Shell

В `shell.c`:
```c
void shell_execute(const char* cmd) {
    if (strcmp(cmd, "mycommand") == 0) {
        // Ваш код
    }
    // ...
}
```

### Добавление системного вызова

1. Добавить константу в `syscall.h`:
   ```c
   #define SYSCALL_MYNEW 5
   ```

2. Обработать в `syscall.c`:
   ```c
   case SYSCALL_MYNEW:
       // Ваш код
       break;
   ```

## Технические детали

### Используемые технологии
- **Язык**: C (C99), Assembly (NASM)
- **Архитектура**: x86 (32-bit, Protected Mode)
- **Загрузчик**: GRUB (Multiboot)
- **Компилятор**: GCC с флагами `-m32 -ffreestanding`
- **Линкер**: GNU LD

### Особенности реализации
- Без стандартной библиотеки C (freestanding)
- Прямая работа с железом (bare metal)
- Кооперативная многозадачность
- Простой аллокатор памяти (first-fit)
- VGA текстовый режим
- PS/2 клавиатура
- Программируемый таймер (PIT)

### Ограничения
- Только 32-bit режим
- Нет файловой системы
- Нет сетевой поддержки
- Нет USB поддержки
- Базовая поддержка процессов
- Нет виртуальной памяти (paging)
