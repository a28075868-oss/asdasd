СПОСОБ 1: WSL2
═══════════════════════════════════════════════════════════════════════════════

1. Откройте Ubuntu (WSL)

2. Перейдите в папку проекта:
   cd /mnt/c/Users/a2807/Desktop/legacyclicker2

3. Очистите старые файлы:
   make clean

4. Пересоберите:
   make

5. Новый entrada.iso готов!

═══════════════════════════════════════════════════════════════════════════════
СПОСОБ 2: GitHub Codespaces
═══════════════════════════════════════════════════════════════════════════════

1. Откройте ваш репозиторий на GitHub

2. Загрузите обновленные файлы:
   - keyboard.c
   - kernel.c
   - screen.c
   - idt.c

3. Откройте Codespace

4. В терминале:
   make clean
   make

5. Скачайте новый entrada.iso

┌──────────────────────────────────────────────────────────────────────────────┐
│ 🐛 ТЕХНИЧЕСКИЕ ДЕТАЛИ ИСПРАВЛЕНИЙ                                            │
└──────────────────────────────────────────────────────────────────────────────┘

ПРОБЛЕМА 1: Клавиатура не работала
ПРИЧИНА: Обработчик прерывания не был зарегистрирован
РЕШЕНИЕ: Добавлен вызов register_interrupt_handler(33, keyboard_handler)
          в функцию keyboard_init()

ПРОБЛЕМА 2: Прерывания не обрабатывались
ПРИЧИНА: Прерывания были отключены (cli)
РЕШЕНИЕ: Добавлен вызов asm volatile("sti") после инициализации всех
          подсистем в kernel_main()

ПРОБЛЕМА 3: Курсор не виден
ПРИЧИНА: Курсор не был включен явно
РЕШЕНИЕ: Добавлена настройка регистров VGA для включения мигающего курсора
          в функции screen_set_cursor()

┌──────────────────────────────────────────────────────────────────────────────┐
│ ❓ ЕСЛИ ВСЕ ЕЩЕ НЕ РАБОТАЕТ                                                  │
└──────────────────────────────────────────────────────────────────────────────┘

Проверьте настройки виртуальной машины:

VirtualBox:
  • Настройки → Система → Включить IO APIC (должно быть ВЫКЛЮЧЕНО)
  • Настройки → Система → Чипсет: PIIX3
  • Настройки → Дисплей → Графический контроллер: VBoxVGA

QEMU:
  • Используйте: qemu-system-i386 -cdrom entrada.iso
  • НЕ используйте -nographic или -curses

VMware:
  • Убедитесь что виртуальная машина имеет фокус
  • Нажмите внутри окна ВМ для захвата клавиатуры

┌──────────────────────────────────────────────────────────────────────────────┐
│ 💡 СОВЕТ                                                                     │
└──────────────────────────────────────────────────────────────────────────────┘

После запуска ОС:
1. Кликните внутри окна виртуальной машины
2. Клавиатура должна быть захвачена автоматически
3. Вы увидите мигающий курсор после приглашения "entrada> "
4. Начните печатать команды

Для выхода из захвата клавиатуры:
- VirtualBox: Правый Ctrl
- VMware: Ctrl + Alt
- QEMU: Ctrl + Alt + G
